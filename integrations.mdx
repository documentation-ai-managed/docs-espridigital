## Overview

Connect Espridigital to your favorite tools to streamline workflows, automate updates, and embed dynamic content. You can sync documentation with Git repositories, set up webhooks for real-time notifications, embed external APIs, and integrate with project management apps like Jira or Asana. These integrations keep your docs always up-to-date without manual effort.

<Callout kind="tip">
  Start with Git integration for version-controlled docs, then add webhooks for CI/CD pipelines.
</Callout>

## Connect to Git Version Control

Link your Git provider to Espridigital for seamless synchronization. Changes in your repo automatically publish to docs.

<Tabs>
  <Tab title="GitHub" icon="github">
    <Steps>
      <Step title="Create App" icon="settings">
        In GitHub, go to Settings > Developer settings > GitHub Apps. Create a new app with repo permissions.
      </Step>
      <Step title="Configure Webhook" icon="link">
        Set webhook URL to `https://api.espridigital.com/v1/webhooks/github`.
      </Step>
      <Step title="Add to Espridigital" icon="plus">
        In Espridigital dashboard, navigate to Integrations > GitHub. Paste your app ID and secret.
      </Step>
    </Steps>

    <CodeGroup tabs="cURL">
    ````bash
    curl -X POST https://api.espridigital.com/v1/integrations/github \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -d '{
        "repo": "your-org/your-repo",
        "branch": "main",
        "webhook_secret": "your-webhook-secret"
      }'
    ````
    </CodeGroup>
  </Tab>
  <Tab title="GitLab" icon="git-branch">
    Follow similar steps using GitLab's project hooks pointing to `https://api.espridigital.com/v1/webhooks/gitlab`.
  </Tab>
</Tabs>

## Webhooks for Automation

Use webhooks to trigger actions like doc rebuilds on events from CI/CD or external services.

### Incoming Webhook Example

Espridigital receives payloads at `https://api.espridigital.com/v1/webhooks/events`.

<Request tabs="Generic,CI/CD">
  ````json
  {
    "event": "page.updated",
    "project_id": "proj_123",
    "payload": {
      "path": "/docs/introduction",
      "content": "Updated content"
    }
  }
  ````

  ````json
  {
    "event": "build.success",
    "pipeline_id": "pipe_456",
    "artifacts": ["docs.zip"]
  }
  ````
</Request>

<ParamField path="event" param-type="string" required="true">
  The webhook event type, e.g., `page.updated` or `build.success`.
</ParamField>

<ParamField header="X-Webhook-Signature" param-type="string" required="true">
  HMAC signature for verification using your webhook secret.
</ParamField>

### Outgoing Webhooks Setup

<Steps>
  <Step title="Enable Outgoing" icon="send">
    Go to Settings > Webhooks > Add Outgoing.
  </Step>
  <Step title="Configure Events" icon="zap">
    Select triggers like `doc.published` or `comment.added`.
  </Step>
  <Step title="Test Endpoint" icon="play">
    Send a test payload to your URL, e.g., Slack or Discord.
  </Step>
</Steps>

## Embed External Content and APIs

Embed live data from third-party services directly into your docs.

Use iFrames for dashboards:

```
<iframe src="https://dashboard.example.com/widget" width="100%" height="400"></iframe>
```

For API embeds, fetch data client-side:

<CodeGroup tabs="JavaScript,React">
  ````javascript
  async function embedData() {
    const response = await fetch('https://api.example.com/metrics?key=YOUR_API_KEY');
    const data = await response.json();
    document.getElementById('embed').innerHTML = JSON.stringify(data, null, 2);
  }
  embedData();
  ````

  ````jsx
  import { useEffect, useState } from 'react';

  function ApiEmbed() {
    const [data, setData] = useState(null);
    useEffect(() => {
      fetch('https://api.example.com/metrics?key=YOUR_API_KEY')
        .then(res => res.json())
        .then(setData);
    }, []);
    return <pre>{JSON.stringify(data, null, 2)}</pre>;
  }
  ````
</CodeGroup>

<Expandable title="Advanced Embedding Options" default-open="false">
  Customize embeds with query params for filtering. Use `{cache: true}` in fetch options to reduce API calls.

  ```mermaid
  sequenceDiagram
      User->>Docs: Load page
      Docs->>External API: Fetch data
      External API->>Docs: Return JSON
      Docs->>User: Render embed
  ```
</Expandable>

## Best Practices

- Verify all webhooks with signatures to prevent spoofing.
- Use environment variables for secrets like `YOUR_API_KEY`.
- Test integrations in a staging project first.

<Callout kind="alert">
  Never expose sensitive tokens in public docs. Use server-side rendering for API keys.
</Callout>
